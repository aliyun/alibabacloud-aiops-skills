# Introduction

The Amazon S3 data source provides you with a bidirectional channel for reading from and writing to Amazon S3. It supports writing TXT/CSV, ORC, and Parquet format files. S3 is an unstructured data source, so you need to ensure that the column structure meets your expectations when writing.

# Destination

## Parameter Description

| Parameter | Description | Required | Default Value | Supported in Wizard Mode | Notes |
|----------|----------|----------|--------|------------------|------|
| datasource | Data source name. Script mode supports adding data sources, and the value of this configuration item must be consistent with the name of the added data source. | Y | None | Y | |
| fileFormat | Text type, the file type to write to S3. | Y | csv | Y | Frontend default value: `csv`; Linkage logic: Switching fileFormat resets writeHeader status and notifies the column mapping component to update |
| object | S3 Object information (filename with path). For example, a.txt or aaa/bbb/cc.csv. | Y | None | Y | Required field |
| column | Column configuration for the output file. For csv/text, use numeric placeholders like `["0", "1"]`; for parquet/orc, configure name and type combinations. | Y | None | Y | Auto-generated by the column mapping component |
| writeMode | Prefix conflict handling strategy. | Y | truncate | Y | Frontend default value: `truncate` (replace existing files); Frontend options: `Replace existing files` → `truncate`, `Keep existing files` → `append`, `Exit with error` → `nonConflict` |
| fieldDelimiter | Field delimiter for writing. | Y when fileFormat is csv/text | , | Y | Display condition: fileFormat is csv or text; If the delimiter is not visible, please use Unicode encoding, such as `\u001b`, `\u007c`; Frontend default value: `,` |
| lineDelimiter | Line delimiter for writing. | Y when fileFormat is text | \n | Y | Display condition: fileFormat is text |
| encoding | Encoding format for the output file. | Y when fileFormat is csv/text | UTF-8 | Y | Display condition: fileFormat is csv or text; Frontend default value: UTF-8 |
| nullFormat | Defines the string that can represent null. For example, when configured as `"null"`, source data `"null"` will be treated as a null value when written. | N | None | Y | |
| dateFormat | Format pattern for date type fields in the output. | N when fileFormat is csv/text | None | Y | Display condition: fileFormat is csv or text |
| writeSingleObject | Whether to write data to a single file (`true`/`false`). | N when fileFormat is csv/text | false | Y | Display condition: fileFormat is csv or text; Not effective for ORC/Parquet formats; Multiple files may still be generated under single concurrency |
| writeHeader | Whether to output the header in the first row (`Yes`/`No`). Header names can be set in column mapping. | N when fileFormat is csv/text | false | Y | Display condition: fileFormat is csv or text; When switched to `Yes`, updateTargetExtend notifies the column mapping component to recalculate; Reset to false when fileFormat is switched |
| header | Array of header names for writing, such as `["id", "name", "age"]`. | N | None | N | Auto-generated by the column mapping component based on writeHeader and field mapping relationships |


## Configuration Example

CSV format write example:

```json
{
  "stepType": "s3",
  "parameter": {
    "datasource": "s3_target",
    "object": "test/csv_file.csv",
    "fileFormat": "csv",
    "encoding": "utf-8",
    "fieldDelimiter": ",",
    "lineDelimiter": "\n",
    "column": ["0", "1"],
    "header": ["col_bigint", "col_tinyint"],
    "writeMode": "truncate",
    "writeSingleObject": true
  },
  "name": "Writer",
  "category": "writer"
}
```

Parquet format write example:

```json
{
  "stepType": "s3",
  "parameter": {
    "datasource": "s3_target",
    "object": "output/data.parquet",
    "fileFormat": "parquet",
    "column": [
      {"name": "id", "type": "long"},
      {"name": "name", "type": "string"},
      {"name": "age", "type": "int"}
    ],
    "writeMode": "append",
    "nullFormat": ""
  },
  "name": "Writer",
  "category": "writer"
}
```
