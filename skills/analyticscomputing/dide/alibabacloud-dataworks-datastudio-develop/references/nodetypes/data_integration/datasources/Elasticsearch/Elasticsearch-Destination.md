# Introduction

The Elasticsearch data source provides you with a bidirectional channel for reading from and writing to Elasticsearch. This article introduces the Elasticsearch data synchronization capabilities supported by DataWorks.

# Destination

## Parameter Description

| Parameter | Description | Required | Default Value | Supported in Wizard Mode | Notes |
|----------|----------|----------|--------|------------------|------|
| datasource | Select the Elasticsearch data source to synchronize. If you have not created this data source in DataWorks, please create it first. | Y | None | Y | |
| index | The index name in Elasticsearch. | Y | None | Y | Linkage logic: checks if the table already exists after selection; clears the type (index type) field when the index changes; prerequisite: index list can be fetched after a successful connection. |
| type | The type name of the index in Elasticsearch. | N | None | Y | Display condition: Displayed when an index is selected and the data source extension info `needElasticSearchType === true` (ES version below 7.0); Linkage logic: depends on the index value to fetch the type list, cleared when the index changes. |
| cleanup | Defines whether the current task should delete data when the index already exists. * Yes (true): Deletes the original index before importing data and recreates an index with the same name. | deleteQueryDSL | Pre-import delete statement (JSON format DSL statement). | N | None | Y | Display condition: Displayed when `cleanup === false` (delete original index not selected); Validation rule: must be valid JSON. |
| actionType | Indicates the action type of Elasticsearch when writing data. Currently, Data Integration supports index and update actionType. The default value is index: * index: Uses the Index.Builder of the Elasticsearch SDK to construct batch requests at the underlying level. If no ID is specified, Elasticsearch will automatically generate a unique ID. If an ID is specified, it will update (replace the entire document). * update: Updates the document based on the user-specified ID. When the action type is set to update, you need to set the primary key **primaryKeyInfo**. | N | index | Y | Frontend options: `Insert` â†’ `'index'`; `Update` â†’ `'update'`; Linkage logic: when "Update" is selected, the primary key is required, otherwise a validation error message is shown. |
| dynamic | Defines whether the synchronization task adds mappings for fields through Elasticsearch's dynamic mapping mechanism when non-existent fields are found in the document. * true: Preserves Elasticsearch's automatic mappings. * false: Default value. | primaryKeyInfo | Defines how the primary key value is determined when writing to Elasticsearch. * **Business Primary Key (pk)**: The _id value is specified as a certain field. * **Composite Primary Key (specific)**: The _id value is specified as a concatenation of several field values, with the separator being the **Primary Key Delimiter** **fieldDelimiter** you set. * **No Primary Key (nopk)**: The _id is automatically generated by the system when writing to Elasticsearch. | Y | specific | Y | Frontend default value: `specific` (Composite Primary Key); Linkage logic: controls the display of fieldDelimiter and column. |
| primaryKeyInfo.fieldDelimiter | Primary key delimiter (used for composite primary key concatenation). | N | None | Y | Display condition: Displayed when `primaryKeyInfo.type === 'specific'` (Composite Primary Key). |

| esPartitionColumn | Defines whether to enable partitioning when writing to Elasticsearch, used to modify the routing parameter in Elasticsearch. * Enable partitioning: The value of the specified column is concatenated with an empty string delimiter as the routing value. * Disable partitioning: If this parameter is not filled in, _id is used as routing by default. | N | None | Y | Display condition: Displayed and required when `enablePartition === true` (partitioning enabled); Data source: dynamically mapped from the destination table structure columns. |
| enablePartition | Whether to enable partitioning. | N | false | Y | Frontend options: `Yes` â†’ `true`; `No` â†’ `false`; Linkage logic: when enabled, esPartitionColumn is displayed. |
| batchSize | Defines the number of Documents inserted into Elasticsearch in a single batch by the synchronization task. | N | 1,024 | Y | Unit: records. |
| timeout | Client timeout. | N | 600,000 | Y | Unit: ms, minimum value: 1. |
| discovery | Whether the task enables node discovery. * true: Connects to a random node in the cluster. | settings | Settings when creating an index, consistent with Elasticsearch's official specification. | N | None | Y | Validation rule: must be valid JSON. |
| column | column is used to configure multiple field information of the document. Each field item can be configured with basic settings such as name and type, as well as extended configurations like Analyzer, Format, and Array. Elasticsearch supported field types include: id, string, text, keyword, long, integer, short, byte, double, float, date, boolean, binary, integer_range, float_range, long_range, double_range, date_range, geo_point, geo_shape, ip, token_count, array, object, nested. | trySize | Defines the number of retries after a write to Elasticsearch fails. | N | 30 | N |  |
| compression | HTTP request, enables compression. | N | true | N |  |
| multiThread | HTTP request, whether multi-threaded. | N | true | N |  |
| ignoreWriteError | Ignore write errors, no retry, continue writing. | N | false | N |  |
| ignoreParseError | Ignore data format parsing errors, continue writing. | N | true | N |  |
| alias | Elasticsearch alias. Configuring alias means creating an alias for the specified index after data import is complete. | N | None | N |  |
| aliasMode | The mode for adding an alias after data import is complete, including append (add mode) and exclusive (keep only this one). | N | append | N |  |
| enableWriteNull | This parameter controls whether null value fields from the source are synchronized to Elasticsearch. | N | true | N |  |

> Y = Yes, N = No

## Configuration Example

```json
{
  "stepType": "elasticsearch",
  "parameter": {
    "datasource": "xxx",
    "index": "test-1",
    "type": "default",
    "cleanup": true,
    "settings": {
      "number_of_shards": 1,
      "number_of_replicas": 0
    },
    "discovery": false,
    "primaryKeyInfo": {
      "type": "pk",
      "fieldDelimiter": ",",
      "column": []
    },
    "batchSize": 1000,
    "dynamic": false,
    "column": [
      {
        "name": "pk",
        "type": "id"
      },
      {
        "name": "col_text",
        "type": "text",
        "analyzer": "ik_max_word"
      }
    ]
  },
  "name": "Writer",
  "category": "writer"
}
```
