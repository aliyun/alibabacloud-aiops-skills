# Verification Method

## 1. Local Prerequisites

```bash
aliyun version
aliyun plugin show --name agentloop
aliyun configure list
```

Expected:

- Aliyun CLI is 3.3.3 or later.
- `aliyun-cli-agentloop` is 0.7.1 or later and supports API `2026-05-20`.
- A usable profile/identity and intended region are shown without exposing credentials.

## 2. Command Surface

Before using an unfamiliar parameter, confirm it appears in help:

```bash
aliyun agentloop <subcommand> --help
```

Validate these six public Dataset subcommands: `create-dataset`, `list-datasets`, `get-dataset`, `update-dataset`, `add-dataset-data`, and `execute-query`. Do not infer unpublished version-management commands.

## 3. Dry Run

Append `--cli-dry-run` to complex JSON, writes, updates, or queries.

Inspect:

- `Endpoint` matches `agentloop.<region>.aliyuncs.com` or the explicitly approved endpoint.
- `API Version` is `2026-05-20`.
- `API Action` matches the intended operation.
- The URL contains the exact AgentSpace and Dataset.
- JSON numbers, booleans, arrays, objects, and null values keep their types.
- A retry uses the same `clientToken` as the original logical request.

Dry run validates serialization only. It does not validate that the resource exists, the identity is authorized, embedding generation can execute, or the SQL can execute.

## 4. Dataset Verification

After create or update:

```bash
aliyun agentloop get-dataset \
  --region <region_id> \
  --agent-space <agent_space_name> \
  --dataset-name <dataset_name>
```

Expected:

- `datasetName` and `agentSpace` match.
- `schema` contains the requested fields and types.
- `description` reflects the requested value.
- `requestId` is present.

For list pagination, verify the requested filter, page size, and `nextToken`; pass the returned token unchanged to the next request.

## 5. Structured Write Verification

Check `affectedRows` equals the number of submitted rows. Then issue a narrow SELECT using a stable supplied `id` or another unique value:

```bash
aliyun agentloop execute-query \
  --region <region_id> \
  --agent-space <agent_space_name> \
  --dataset-name <dataset_name> \
  --type SQL \
  --query "SELECT id, __time__, <fields> FROM <dataset_name> WHERE id = '<row_uuid>' LIMIT 1"
```

If IDs were generated by the service, query the newly written records using a narrow unique predicate and time window.

## 6. Query Verification

Expected for SELECT/search:

- `meta.progress` is `Complete`.
- `meta.count` equals the number of returned rows.
- `columns` and `columnTypes` have matching lengths.
- Every row has the same number of entries as `columns`.
- If `maxOutputLength` was used, inspect `meta.truncation` before treating values as complete.

## 7. Observability Verification

Confirm that every `aliyun agentloop` invocation, including dry runs and verification calls, used the same session-scoped user agent:

```bash
--user-agent "AlibabaCloud-Agent-Skills/alibabacloud-agentloop-dataset/${SESSION_ID}"
```

`SESSION_ID` must be the single 32-character lowercase hexadecimal value generated once for this skill session. Do not regenerate it for retries or mutate global Aliyun CLI configuration to set the user agent.
