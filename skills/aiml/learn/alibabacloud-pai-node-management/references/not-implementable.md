# NOT Implementable via Public CLI/SDK — PAI Node

The following Node-related capabilities are **NOT exposed** through the public `aliyun paistudio` plugin or the public PAI OpenAPI (cross-check at https://next.api.aliyun.com/document/PaiStudio/2022-01-12/overview). They cannot be invoked from this skill. Cordon / Uncordon / Drain **are** in scope via `aliyun paistudio operate-node` under the SKILL.md §6.4 confirmation gate; everything else below is out of scope.

| Capability | Where it actually lives / Workaround |
| --- | --- |
| Node power (start / stop / reboot) | **NOT supported by this skill — refuse.** The agent MUST NOT execute or redirect to the underlying ECS instance's power-state CLI / SDK (whether kebab-case `aliyun ecs …` or legacy CamelCase actions), even when the user can name the ECS instance ID. Power-state mutation of a PAI-managed node is restricted to the platform Ops team — escalation is **three-way only**: platform Ops team / Lingjun ops ticket / account manager. **[Refusal-text forbidden-string rule]** The agent's refusal text MUST NOT contain the literal command names — see the *Forbidden literal-string rule for node power / reboot refusals* callout in `SKILL.md` §1. Describe the forbidden path in plain English ("the underlying instance's power-state CLI") instead of echoing it. |
| `DrainParameters.Force` (force-drain a stuck Pod) | NOT exposed by this skill. Backend Ops allowlist only. The agent MUST NOT include `"Force"` in `DrainParameters` and MUST NOT suggest `kubectl drain --force` / SSH `kill -9` as substitutes. Three-way escalation: platform Ops team / Lingjun ops ticket / account manager. |
| `GetNode` / `GetNodeGPUMetrics` / `ListNodePods` (per-node metrics & pod listing) | NOT in public `aliyun paistudio` CLI. Direct the user to the **PAI Console** ResourceGroup / Quota aggregate dashboards. **Forbidden** as a fallback in the reply: `nvidia-smi` (whether via SSH, `kubectl exec`, or otherwise), `kubectl top node`, `kubectl describe node`, `kubectl get pods --field-selector spec.nodeName=...`, Prometheus / DCGM-exporter direct query, SSH-into-node. If node-level data is genuinely required, three-way escalation: platform Ops team / Lingjun ops ticket / account manager. |
| Node replacement / migration | NOT exposed. Three-way escalation: platform Ops team / Lingjun ops ticket / account manager. |
| Node label / taint mutation | NOT exposed. Cluster-internal control-plane operation; **forbidden** to suggest `kubectl label node` / `kubectl taint node` as a workaround. |
| Node attach / detach to a MachineGroup | Purchase-time only (`CreateMachineGroup` BSS flow). Not exposed via `paistudio`. |
| Cross-account node sharing | Not exposed publicly. |
| Real-time per-pod accounting on a node | Use the `pai-quota-management` skill's `list-quota-workloads` instead. **Forbidden** to suggest `kubectl get pods --field-selector spec.nodeName=...` as a substitute. |

> **Rule**: if a capability is not in the public OpenAPI portal or in the `aliyun paistudio --help` output, treat it as **not implementable** by this skill. Do NOT fall back to SDKs (Python / Java / Go) / `curl` / MCP / Terraform / `kubectl` (any subcommand) / SSH / `nvidia-smi` / any other node-level direct-access. Nodes are platform-managed resources and not directly reachable by the user or the agent — even when the user explicitly asks for one of these tools, the agent MUST refuse and MUST NOT echo back the command. If the user needs node mutation or node-level data, escalate via the **three-way** path: platform Ops team / Lingjun ops ticket / account manager (outside this skill).
